Getting the Most Out of KissMT Dynamic SEO Meta Tags Base Install

Settings

catalog/includes/modules/kiss_meta_tags/includes/init.php

  • KISSMT_CATEGORIES_PRODUCTS_LIMIT - numeric - the number of categories/products to pull from the database in order to provide a reasonable length description.
  • KISSMT_TITLE_SEPARATOR - string - the character that will separate the words in your browser title .. like my | website or my - website
  • KISS_MT_META_DESCRIPTION_LENGTH - INT - Limit the length of the meta description.
  • KISS_MT_META_KEYWORDS_NUMWORDS - INT - Limit the number of meta keywords.
  • KISS_MT_META_KEYWORD_SIZE - INT - Set the number of words in each generated keyword (product).
  • KISS_MT_META_TITLE_LIMIT_NUMWORDS - INT - Limit the product name length.
  • KISS_MT_META_TITLE_NUMWORDS - INT - Limit the product name to X words in meta title if the above is true.
  • KISS_MT_META_TITLE_LENGTH - INT - Limit the overall meta title length
  • KISS_MT_PAGE_TITLE_LENGTH - INT - Limit the meta title to the nearest full word to X characters.
  • KISSMT_XHTML_OUTPUT - string - true/false - is your site xhtml? set to true will output xhtml tags
  • KISSMT_PERFORMANCE_OUTPUT - string true/false - outputs KissMT performance data at the bottom of the page.
  • KISSMT_CLASS_OUTPUT - string true/false - outputs the properties of the KissMT class, as long as KISSMT_PERFORMANCE_OUTPUT is also set to true
  • KISSMT_CACHE_RESET - string reset/false - resets the cache on the next front end page load. (Has to be set back to false manually)
  • KISSMT_CACHE_ON - string true/false - whether to cache the data or run queries.
  • KISSMT_CANONICAL_ON - string true/false - output canonical tags to remove duplicate search engine index entries and osCsid sessions

Database Keywords

Keywords stored in the product description table for each product will take precedence over generated keywords.

I see text in the title but don't know how to change it

You'll see text in the title like "Products by Some Manufacturer at My Shop"

These are set in the language files .. catalog/includes/languages/[my language]/kissmt.php

What is the %s

In the language file the %s means that the code is going to replace it with some text. Play about with it .. if the text is "Buy securely online at %s" try changing it to "%s buy securely online.
Once you understand what it does it's very simple and flexible."

The main files have propper title/description/keywords but (some other file) doesn't .. how do I add it?

You can add propper title/description/keywords to any file in the catalog/shop root directory very simply.

Look in the language file (kissmt.php) for KISSMT_PRIVACY_TITLE_TEXT.
You should see that there are two entries related to privacy .. KISSMT_PRIVACY_TITLE_TEXT and KISSMT_PRIVACY_DESCRIPTION_TEXT.

If you ever want to add propper title/description/keywords to a new file you just have to create these two entries in the language file (you obviously have to have replaced the title /title code with the KissMT code in the main file) .. as an example ..

Let's say that you want to add this for a page called my_new_page.php. You would need to create the following two entries as defines in the language file ..

  • KISSMT_MY_NEW_PAGE_TITLE_TEXT .. e.g. define( 'KISSMT_MY_NEW_PAGE_TITLE_TEXT', 'New - Page - Title' );
  • KISSMT_MY_NEW_PAGE_DESCRIPTION_TEXT .. e.g. define( 'KISSMT_MY_NEW_PAGE_DESCRIPTION_TEXT', 'This is my new page' );

See how the capitalised MY_NEW_PAGE from my_new_page.php replaced PRIVACY?
The script will automatically recognise this to create the title/description and will also create the keywords for you automatically.

What are Stopwords

Text from http://www.seoglossary.com/article/120

Certain words, such as "the," "a", "an," "of," and "with," are so common and meaningless that a search engine won't bother including them in their index, or database, of web page content. So in effect, the stop words on your web pages are ignored as if those words weren't on your pages in the first place. Including a lot of stop words in your title tag waters down the title tag's keyword density.

Where are the stopwords?

The stopwords can be found at catalog/includes/modules/kiss_meta_tags/stopwords/ and are named using your osCommerce language.
e.g. The stopwords file for the english language is named .. english_stopwords.ini

You will find the language based stopwords in extras/stopwords. Please ensure that the naming of the stopwords file matches your osCommerce language.

My language is not English and there is no stopwords file available

We will try to include the basic stopwords of the main European languages as best as we can, but they will all be limited.
My suggestion is to search in your local language on Google for "seo stop words [mylanguage]" and "seo stopwords [my language]".
If you look well enough there MUST be an seo company/organisation listing them .. when you find them you can create your own stopwords file, please forward these to me so others can benefit.
